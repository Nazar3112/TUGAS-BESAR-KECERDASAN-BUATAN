Algoritma Aplikasi Klasifikasi Hoaks MBG Menggunakan ID3

1. Struktur Program Utama

START PROGRAM

Inisialisasi:
    dataset ← kosong
    model ← kosong
    label_encoder ← kosong
    feature_names ← kosong

Tampilkan GUI:
    - Tombol Load Dataset
    - Tombol Train Model
    - Input Judul
    - Tombol Prediksi

Menunggu interaksi user

END PROGRAM

2. Load Dataset

FUNCTION load_dataset()

    Pilih file (.csv / .xlsx)

    IF file tidak dipilih → RETURN

    Baca file ke dalam dataset

    IF tidak ada kolom "label"
        Tampilkan error
        RETURN

    IF tidak ada kolom fitur DAN tidak ada "judul_post"
        Tampilkan error
        RETURN

    Simpan dataset
    Tampilkan pesan berhasil

END FUNCTION

3. Train Model (Proses ID3)

FUNCTION train_model()

    IF dataset kosong → RETURN

    Ambil fitur (X) dan label (y)

    Encode label menjadi angka

    Bangun Decision Tree dengan kriteria ENTROPY

    Proses ID3:
        Hitung entropy awal
        Untuk setiap atribut:
            Hitung entropy tiap cabang
            Hitung Information Gain
        Pilih atribut dengan IG terbesar sebagai root
        Ulangi proses pada setiap cabang
        Bentuk leaf node (Fakta / Hoaks)

    Simpan model
    Tampilkan feature importance

END FUNCTION

4. Prediksi Judul Baru

FUNCTION predict()

    IF model belum dilatih → RETURN

    Ambil teks judul dari user

    Ekstraksi fitur:
        - ada_data_bukti
        - sumber_resmi
        - kata_provokatif
        - klaim_berlebihan

    Bentuk data input sesuai urutan fitur

    Lakukan prediksi dengan model

    Ambil probabilitas (confidence)

    Tampilkan:
        - Hasil (Fakta / Hoaks)
        - Confidence
        - Fitur yang terdeteksi

END FUNCTION

5. Pseudocode Konsep Matematis ID3

FUNCTION ID3(dataset, atribut)

    IF semua data satu label
        RETURN leaf(label)

    IF atribut kosong
        RETURN leaf(label mayoritas)

    Hitung entropy dataset

    Untuk setiap atribut:
        Hitung Information Gain

    Pilih atribut dengan IG terbesar

    Untuk setiap nilai atribut:
        Buat cabang
        Panggil ID3(subset, atribut tersisa)

    RETURN node
END FUNCTION

Penjelasan Dengan Bahasa Sederhana

1. Load Dataset
Aplikasi membaca data berita yang sudah diberi label Fakta atau Hoaks.

2. Train Model
Program menghitung tingkat ketidakpastian (entropy). Semakin tidak jelas perbedaannya, entropy tinggi.

Lalu program mencoba setiap atribut untuk melihat mana yang paling membantu memisahkan Fakta dan Hoaks. Pengukuran ini disebut Information Gain.

Atribut dengan Information Gain terbesar dijadikan pertanyaan pertama (root pohon).

Proses ini diulang sampai semua data terpisah dengan jelas.

3. Prediksi
Ketika user memasukkan judul baru, sistem:

  • Mengecek apakah ada data/bukti
  • Mengecek apakah ada sumber resmi
  • Mengecek kata provokatif
  • Mengecek klaim berlebihan
Lalu pohon keputusan menentukan hasil akhirnya.

Confidence menunjukkan seberapa yakin model terhadap hasil tersebut.
